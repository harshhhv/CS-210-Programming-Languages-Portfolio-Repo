For my portfolio submission, I chose to include my Project Three, the Corner Grocer item-tracking program. This project focused on analyzing a text file containing a list of purchased grocery items and determining how frequently each item appeared. The main problem it solved was helping a business understand customer purchasing patterns so they could make better decisions about store layout and product placement. The program reads data from a file, processes it using a map data structure, and provides multiple outputs such as item frequency, search functionality, and a visual histogram representation.

One thing I believe I did particularly well in this project was structuring the program using object-oriented principles. I created a dedicated class to handle all data processing and separated it from the main function, which kept the code organized and easier to follow. I also made sure to use clear naming conventions and include comments throughout the code, which improves readability and makes it easier for someone else to understand the logic without needing to trace every line.

That said, there are definitely areas where I could improve. One major improvement would be adding stronger input validation and error handling. For example, handling invalid user input more gracefully or making the search feature case-insensitive would make the program more robust. I could also enhance efficiency by sorting items based on frequency or optimizing how data is displayed for larger datasets. These changes would make the program more scalable and closer to real-world application standards.

The most challenging part of this project was working with file input/output and ensuring the data was read and processed correctly. Handling file paths in Visual Studio and making sure the input file was accessible at runtime took some trial and error. I overcame this by understanding how the program’s working directory functions and using project settings like “Copy to Output Directory.” I also relied on documentation and debugging to track where things were going wrong, which helped build my confidence in troubleshooting.

This project helped strengthen several skills that are transferable to other work, including working with file handling, using data structures like maps, and designing modular programs using classes. These are foundational skills that apply to many areas, from backend systems to data processing pipelines. It also improved my ability to think about how data flows through a system rather than just focusing on individual lines of code.

To make the program maintainable and adaptable, I focused on separating responsibilities within the code. The class handles all data-related operations, while the main function acts as a controller for user interaction. This makes it easier to modify or extend specific parts of the program without affecting the entire system. Overall, this project gave me a better understanding of how to write cleaner, more structured code that can scale beyond simple assignments.
